# Spec: Save/Load v2 — Full Mid-Level State

## Context

The current save/load (`Z` saves, `X` loads) only captures 6 fields:
score, level, lives, start_level, paddle_size, num_bullets. On load,
the level reloads fresh from `levelNN.data` and ball/specials/eyedude
reset. This makes save/load redundant with the `w` key (jump to level
N fresh).

Goal: capture full mid-level state so save/load (a) actually resumes
mid-level for gameplay and (b) serves as a test fixture mechanism —
write a save file, load it, drive from that exact point.

## Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| File scheme | Two files | Matches `original/file.c`; `paths_save_level` already declared; grid is naturally text-shaped |
| Block grid format | Full-fidelity JSON | Preserves `random`, BLACK_BLK `hit_points`/`next_frame`, resolved RANDOM_BLK types |
| v1 compatibility | None | No v1 saves in the wild |
| RNG seed | Not saved | Fixtures are state-setup, not exact-replay |
| Mid-explosion blocks | Dropped | Reset to non-exploding on load |

## Scope

### In scope

- `save-info.dat` (JSON): all player + game state + active specials + per-ball state + eyedude + paddle position + bonus_count + user_tilts + time_remaining + gun_unlimited
- `save-level.dat` (JSON): per-cell block grid state (type, counter_slide, random flag, hit_points, BLACK_BLK next_frame offset)
- Auto-save at bonus screen wired up (`bonus_cb_on_save_triggered` TODO completed)
- `level_time`/`time_remaining` bugs fixed
- Test fixture API: `savegame_system_capture` / `savegame_system_restore` as pure functions

### Out of scope

- In-flight bullets (reset on load, matches original)
- Mid-explosion block animation state (reset)
- Sub-second timer accumulator (resets)
- Visual SFX state (shake, fade, glow, sparkles)
- Audio queue
- Exact RNG replay

## Schemas

### `save-info.dat`

```json
{
  "version": 2,
  "score": 125000,
  "level": 7,
  "level_time": 180,
  "time_remaining": 142,
  "game_time": 14400,
  "lives_left": 3,
  "start_level": 1,
  "user_tilts": 1,
  "bonus_count": 3,
  "paddle_pos": 247,
  "paddle_size_type": 5,
  "paddle_reverse": 0,
  "paddle_sticky": 0,
  "num_bullets": 6,
  "gun_unlimited": 0,
  "specials": {
    "sticky": 0, "saving": 0, "fast_gun": 1, "no_walls": 0,
    "killer": 0, "x2": 0, "x4": 0
  },
  "eyedude": {
    "state": 3, "dir": 1, "x": 120, "y": 16,
    "slide": 2, "inc": 5, "turn": 0
  },
  "balls": [
    {"active": 1, "state": 1, "x": 247, "y": 520, "dx": 3, "dy": -4, "wait_mode": 7},
    {"active": 0, "state": 6, "x": 0, "y": 0, "dx": 0, "dy": 0, "wait_mode": 7}
  ]
}
```

Notes:

- `time_remaining` is the live countdown (was missing from v1 — bug fix)
- `level_time` is the level's starting timer (informational, matches original `saveGameStruct.levelTime`)
- `paddle_size_type` is the enum (`PADDLE_SIZE_SMALL/MEDIUM/HUGE`)
- `balls[]` is exactly `MAX_BALLS` (5) entries, inactive slots have `active: 0`
- All frame-derived fields (eyedude slide, BLACK_BLK next_frame) saved as offsets from save frame, not absolute

### `save-level.dat`

```json
{
  "version": 1,
  "title": "Level 7: The Gauntlet",
  "time_bonus": 180,
  "cells": [
    {"r": 0, "c": 1, "type": 2, "counter_slide": 0, "random": 0, "hit_points": 0, "next_frame_off": 0},
    {"r": 3, "c": 2, "type": 8, "counter_slide": 3, "random": 0, "hit_points": 0, "next_frame_off": 0},
    {"r": 4, "c": 0, "type": 29, "counter_slide": 0, "random": 0, "hit_points": 2, "next_frame_off": 1540}
  ]
}
```

Notes:

- Sparse — only occupied cells included
- `type` is `block_type` integer (matches `block_types.h`)
- `counter_slide` set for COUNTER_BLK (0-5), 0 for others
- `random` preserves RANDOM_BLK semantics on round-trip
- `hit_points` for BLACK_BLK damage state
- `next_frame_off` is BLACK_BLK cooldown offset from save frame (positive = still cooling, 0 = hittable)
- Mid-explosion blocks saved as their original type (animation state dropped)

## Critical Implementation Details

**Frame-relative timestamps:** All saved frame values are converted to
offsets from the save frame. On restore, the loader adds the current
frame counter. Negative offsets clamp to 0 (already-expired states
fire immediately on next tick). Applies to: BLACK_BLK `next_frame`.

**Ball state machine on restore:** `ball_system_restore` sets fields
directly. `nextFrame` and `waitingFrame` default to 0 — state machine
reinitializes on first update. BALL_CREATE saves are restored as
BALL_READY (skip the spawn animation). BALL_ACTIVE / BALL_READY /
BALL_WAIT preserve velocity and position.

**Auto-save level file:** When the bonus screen triggers auto-save,
the level is cleared. Writing `save-level.dat` would produce an
empty-grid file, which on load would immediately trigger level
completion. Fix: `savegame_system_autosave` writes only
`save-info.dat` and deletes any stale `save-level.dat`. The load
path falls back to canonical `.data` file when `save-level.dat` is
missing.

**Specials restore order:** Call `special_system_turn_off` first (which
preserves `SPECIAL_SAVING` per legacy), then explicitly set each of
the 7 flags from the save. The wall-state changed callback fires
when no_walls toggles — already correct.

**Two-file atomicity:** Two-file scheme is not atomic. Mitigation:
write level file first, then info file. Load reads info first; if
the level file is missing or unreadable, fall back to canonical
level. A partial save (info written, level write failed) leaves the
player with a working but slightly-stale state on load.

## Test Strategy

### Unit tests (`tests/test_savegame_io.c`)

- v2 round-trip: write `savegame_data_t` with all fields populated, read back, assert equality
- Level round-trip: write `savegame_level_t` with mix of block types + BLACK_BLK cooldown + COUNTER_BLK + RANDOM_BLK, read back, assert all fields equal
- Missing optional fields tolerated (unknown keys skipped in v1 reader behavior, preserved in v2)
- Atomic write: writer creates temp + renames; partial-write recovery

### System tests (`tests/test_savegame_system.c`)

- `capture/restore` round-trip without I/O: build a populated ctx, capture, restore into fresh ctx, assert state matches
- Test fixture pattern documented for reuse: `make_fixture_at_level(int n, ...)` helper writes save files, loader reaches that state

### Integration test (`tests/test_keybindings.c`)

- Inject Z, verify both save files exist on disk
- Inject X on fresh ctx with known save files, verify ctx fields restored

### Manual verification

- Z mid-level → exit → relaunch → X → verify paddle at saved position, ball moving with saved velocity, killer mode active if it was, BLACK_BLK still damaged
- Complete level 5 → verify auto-save fired → restart → X → resume at level 5 bonus screen

## Risks

- **Two-file partial write**: mitigated by load order + fallback (described above)
- **JSON array parsing in hand-rolled parser**: needs care for `balls[]` and `cells[]`; existing parser doesn't handle arrays — must add
- **State machine assumptions on restore**: BALL_CREATE save dropped to BALL_READY; documented
- **Block animation re-phasing**: RANDOM_BLK/ROAMER_BLK animations restart on load (acceptable, cosmetic only)
